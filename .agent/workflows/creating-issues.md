---
description: When creating new issues on GitHub
---

Use the feature or bug template in the `.github` folder — see
[`AGENTS.md` → Creating an issue](../../AGENTS.md#creating-an-issue).

Beyond the template:

1. Create the issue with high clarity. Describe exactly what it aims to accomplish in a short paragraph.
2. Reopen the issue to add necessary attributes.
3. Ensure the issue appears correctly on the **Roadmap** — this is where most planning time should be spent.
4. Keep the issue up to date with its attributes throughout the development process.
